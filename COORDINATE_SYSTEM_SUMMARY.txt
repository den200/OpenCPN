Level 5: SCREEN PIXEL (Absolute viewport coordinates)
  ├─ Formula: x = pix_width/2 + dx

┃ PRECISION STRATEGY: 3-LEVEL HIERARCHY                                              ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

Why needed? Float32 has ~7 decimal digits precision. Large coordinates lose precision.

Example: Montreal area, 100 km² viewport
  Geographic: 45.5°, -73.6° (±0.05° = ±5.5 km)
  Absolute Mercator: 8,213,461 m (loses sub-meter precision in float)
  Relative to viewport: ±50,000 m (maintains ~0.01 mm with double)
  Pixels: ±2,500 px (perfect integer precision)

Three-level preservation:
  Level 1: Store geographic center in double (clat, clon)
  Level 2: Compute SM relative to chart ref point (m_easting_vp_center)
  Level 3: Offset from viewport center → small values maintain precision

Result: Sub-millimeter accuracy over large charts using 64-bit doubles

┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

Mercator Cache (10x speedup for 1000+ geometry points):
  
  Expensive: y30 = 0.5 * log((1 + sin(lat0)) / (1 - sin(lat0))) * z
  
  Cached approach:
    toSMcache_y30(lat0) [called once at viewport update]
      → precomputes y30
    
    toSMcache(lat, lon, cached_y30, ...) [called per point]
      → y3 = 0.5 * log((1 + sin(lat)) / (1 - sin(lat))) * z
      → y = y3 - cached_y30
  
  Benefit: Avoid repeated sin/log at viewport center latitude

Call frequency:
  SetVPParms() [s57chart.cpp:590]
    ├─ Called: every viewport change (pan/zoom)
    ├─ Executes: one toSM() call

  GetPointPix() [s57chart.cpp:546]
    ├─ Called: per geometry point during rendering
    ├─ Executes: simple arithmetic (no trig)
    └─ Uses: cached m_easting_vp_center values
